# Contributing to Tutorial GitHub

Terima kasih sudah tertarik untuk berkontribusi! Repositori ini adalah community learning resource untuk Git dan GitHub.

## 📋 Jenis Kontribusi yang Kami Welcome

### 1. 📝 Content Improvements
- Memperbaiki typo atau grammar
- Memperjelas penjelasan di modules
- Adding missing examples
- Updating outdated information
- Improving clarity dan accessibility

### 2. 🐛 Bug Reports
- Broken links
- Incorrect git commands
- Outdated GitHub features
- Typos di learning materials

### 3. 💡 Feature Suggestions
- New module ideas
- Playground scenario suggestions
- Project improvement ideas
- Interactive exercise ideas

### 4. 🌍 Translations
- Translate modules ke bahasa lain
- Community translations welcome

### 5. 📚 New Examples
- Real-world workflow examples
- Code samples improvements
- Better diagram atau visualisasi

## 🚀 How to Contribute

### Step 1: Fork & Clone
```bash
git clone https://github.com/<your-username>/tutorial-github.git
cd tutorial-github
```

### Step 2: Create Feature Branch
```bash
git checkout -b feature/describe-your-change
# Examples:
# feature/fix-typo-module-1
# feature/add-conflict-scenario
# feature/clarify-git-flow
```

### Step 3: Make Changes
- Edit files sesuai contribution type
- Test changes locally
- Ensure links working
- Check formatting consistency

### Step 4: Commit dengan Clear Message
```bash
git commit -m "type: description

Longer explanation jika diperlukan.

Closes #123 (jika fix related issue)"
```

**Commit types:**
- `docs:` - Documentation/content changes
- `fix:` - Bug fixes
- `feat:` - New features (new scenarios, modules)
- `refactor:` - Reorganizing content
- `style:` - Formatting, consistency

### Step 5: Push & Create PR
```bash
git push origin feature/describe-your-change
```
Buka PR via GitHub dengan clear description:
- **What**: Apa yang diubah
- **Why**: Mengapa perlu diubah
- **How**: Bagaimana cara reviewnya

## 📐 Content Guidelines

### Module Structure
Setiap module HARUS mengikuti struktur ini:
1. **Konsep** - Teori dan penjelasan
2. **Visualisasi** - Diagram ASCII atau images
3. **Hands-On** - Step-by-step exercise
4. **Real-world** - Contoh dari production projects
5. **Troubleshooting** - Common issues dan solutions

### Writing Style
- **Language**: Bahasa Indonesia (mix dengan English terms untuk Git commands)
- **Tone**: Friendly, accessible untuk junior developers
- **Clarity**: Prefer simplicity over complexity
- **Examples**: Include practical, tested examples
- **Length**: Keep modules concise (~30-60 min read)

### Code Formatting
- Use markdown code blocks dengan syntax highlight
- Git commands: ```bash
- Include comments explaining each step
- Test commands before submitting

### Links
- Use relative links untuk internal resources: `[Module 1](docs/01-pembuatan-repositori/)`
- Use absolute URLs untuk external links

## ✅ PR Review Checklist

Before submitting PR, ensure:
- [ ] Content is clear dan accessible untuk junior developers
- [ ] All links tested dan working
- [ ] No typos atau grammar issues (Indonesian atau English)
- [ ] Code examples tested dan working
- [ ] Diagrams/visualizations clear
- [ ] Follows module structure guidelines
- [ ] Commit messages descriptive
- [ ] No large binary files (keep repo lightweight)

## 📝 Reviewer Guidelines

When reviewing contributions:
- Be constructive dan encouraging
- Suggest improvements rather than rejecting
- Ask questions if unclear
- Test code examples jika ada
- Approve jika meets quality standards
