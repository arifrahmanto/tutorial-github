# 🚀 Git & GitHub Learning Hub

Repositori pembelajaran komprehensif untuk **developer baru** yang ingin menguasai Git, GitHub, dan kontribusi open source.

---

## 📚 Learning Path Overview

Repository ini dirancang sebagai **project-based learning platform** dengan 4 learning projects yang progressif, 8 modul terstruktur, dan 5 playground scenarios untuk hands-on practice.

### ✨ Fitur Utama

- 📖 **8 Modul Pembelajaran Terstruktur**: Dari fundamental hingga advanced
- 🎮 **5 Playground Scenarios**: Scenario reproducible untuk practice berulang
- 📦 **4 Learning Projects**: Escalating complexity dari solo → team → OSS
- 📊 **Progress Tracking**: Checklist per modul dan project
- 🌐 **GitHub Pages**: Static site dengan visualisasi dan tutorial
- 🚀 **GitHub Actions + Netlify**: Automated deployment pipeline

---

## 🎯 Target Audience

Repositori ini dibuat untuk **developer baru** yang ingin:
- ✅ Memahami konsep dasar Git dan GitHub
- ✅ Belajar workflow kolaborasi tim
- ✅ Berkontribusi ke open source projects
- ✅ Menguasai strategi Git dalam software development

## 🎮 4 Learning Projects

Setiap project membangun skill incrementally dengan realistic scenarios.

### Project 1: First Repository
**Tujuan:** Buat repository pertama dan push ke GitHub  
**Durasi:** ~30 menit  
📍 [Mulai Project 1](sample-projects/01-first-repo/)

**Checklist:**
- [ ] Create repository lokal
- [ ] Configure git user
- [ ] Make first commit
- [ ] Connect ke GitHub
- [ ] Push ke remote

---

### Project 2: Team Collaboration
**Tujuan:** Simulasi kolaborasi tim dengan branch, PR, dan merge  
**Durasi:** ~1.5 jam  
📍 [Mulai Project 2](sample-projects/02-team-project/)

**Checklist:**
- [ ] Create feature branches
- [ ] Simulate team changes
- [ ] Create PR dengan description
- [ ] Code review process
- [ ] Merge dan delete branch

---

### Project 3: Conflict Resolution
**Tujuan:** Encounter dan resolve merge conflicts dengan berbagai strategi  
**Durasi:** ~1 jam  
📍 [Mulai Project 3](sample-projects/03-conflict-resolution/)

**Checklist:**
- [ ] Create diverging branches
- [ ] Trigger merge conflict
- [ ] Understand conflict markers
- [ ] Resolve manually
- [ ] Learn prevention strategies

---

### Project 4: Open Source Contribution
**Tujuan:** Fork repository dan contribute dalam OSS style  
**Durasi:** ~1.5 jam  
📍 [Mulai Project 4](sample-projects/04-oss-style/)

**Checklist:**
- [ ] Fork repository
- [ ] Setup upstream remote
- [ ] Create feature branch
- [ ] Push dan create PR
- [ ] Iterate dari feedback

## 🎯 Playground Scenarios

Clone repositori ini dan practice dengan 5 reproducible scenarios. Setiap scenario bisa di-reset dan di-try lagi tanpa merusak materi.

### Scenario 1: First Commit
**Branch:** `scenario-1-first-commit`

Learn basic Git workflow: create file, stage, commit, push.

```bash
git clone <repo-url>
git checkout scenario-1-first-commit
# Follow README instructions
```

---

### Scenario 2: Branching & Merging
**Branch:** `scenario-2-branching`

Practice feature branch workflow, merge ke main, understand fast-forward.

---

### Scenario 3: Merge Conflict
**Branch:** `scenario-3-merge-conflict`

Intentional conflict setup untuk learn resolution strategies.

---

### Scenario 4: Team Collaboration
**Branch:** `scenario-4-collaboration`

Simulate dua developer dengan overlapping changes dan PRs.

---

### Scenario 5: GitHub Actions
**Branch:** `scenario-5-github-actions`

Trigger Actions workflow dan observe deployment.

## ❓ FAQ

**Q: Berapa lama untuk complete semua modules?**  
A: ~8 jam untuk semua modules + projects. Bisa dibuat lebih cepat atau lambat sesuai pace Anda.

**Q: Saya sudah tahu basics Git, mulai dari mana?**  
A: Jump ke [Module 3](docs/03-manajemen-cabang/) atau langsung ke [Project 2](sample-projects/02-team-project/).

**Q: Playground scenarios bisa di-reset?**  
A: Ya! Setiap scenario bisa di-reset dengan `git reset --hard origin/scenario-*-branch` untuk try lagi.

**Q: Ada video tutorials?**  
A: Not yet - kami fokus pada learning materials. External videos bisa dicari untuk supplement.
